ポリシー設定と判定記録

組織のルールに基づいてOSS利用を審査する（v1.1.0）

概要

Dependa のポリシー機能は、OSS の検出結果に対して「組織としてどう判断するか」を設定し、その判定理由を記録・共有するための仕組みです。

単なる検出ではなく、なぜその判定になったかを説明できるレポートを生成します。

ポリシー設定（Policy Builder）

左メニューの「ポリシー」から設定画面を開きます。

ステップ 1: テンプレートを選ぶ

「テンプレートを適用」をクリックすると、選択したテンプレートのルールがプレビューに反映されます。

テンプレート説明
Balanced（バランス型）一般的なリスクバランスを想定した初期設定。多くのプロジェクトに推奨
Strict（厳格型）コピーレフト・不明ライセンスを厳しく制限
Permissive（柔軟型）許容範囲を広げた設定。社内ツール向け

ステップ 2: ルールをカスタマイズ

各ルールのアクションを変更できます。

ルールカテゴリ説明
強いコピーレフトライセンスGPL / AGPL など。ソースコード公開義務がある
弱いコピーレフトライセンスLGPL / MPL など。ライブラリとして分離すれば利用可能な場合が多い
不明なライセンスライセンスが特定できなかったパッケージ
Critical 脆弱性CVSS 9.0 以上の重大な脆弱性
High 脆弱性CVSS 7.0〜8.9 の既知の脆弱性
信頼度の低いライセンス判定検出されたライセンス情報が不完全または不一致
ライセンス情報の欠損ライセンスのメタデータが見つからなかったパッケージ

各ルールに設定できるアクション:

アクション意味色
使用不可組織のポリシーで利用を推奨しない赤
要レビュー利用前に確認が必要橙
条件付き許可条件を満たせば利用可能青
許可利用に問題なし緑

ステップ 3: 保存と既定設定

  1. 「ポリシー名」を入力して「保存」をクリック
  2. 「既定に設定」をクリックすると、次回以降のスキャンでこのポリシーが自動的に適用されます

ポリシーが1件だけ保存されている場合は、「既定に設定」をしなくても自動的にスキャン時に適用されます。

ポリシー判定記録

スキャンを実行すると、各パッケージに対してポリシー判定記録が自動生成されます。

判定記録の5列

列説明
パッケージ対象パッケージ名
検出事実検出された事実（例: ライセンス: 承認、中 — CVE-2024-xxxx）
初期評価既定ルールによる自動評価（OK / 警告 / 不可）
ポリシー判定組織ポリシーによる判定（許可 / 条件付き / 要レビュー / 使用不可 / 制限なし）
理由判定の根拠（例: 不明ライセンス: 要レビューとする設定）

ポリシー判定列の読み方

表示意味
使用不可ポリシーで利用を推奨しないと設定されている
要レビュー利用前に確認が必要と設定されている
条件付き許可条件を満たせば利用可能と設定されている
許可利用に問題なしと設定されている
—（制限なし）ポリシーは適用されているが、このパッケージに該当するルールがない。特に制限なし
—（ポリシー未設定）組織ポリシーが設定されていない

HTML レポートでの表示

スキャン結果から生成される HTML レポートにも判定記録が含まれます。

  • 要約: 適用ポリシー名、アクション別件数バッジ
  • 詳細テーブル: パッケージごとの事実・判定・理由

このレポートはオフラインで閲覧でき、チームメンバーやレビュー担当者と共有できます。

CSV でのポリシー情報

CSV エクスポート（v2 スキーマ）には以下の列が追加されています:

列名説明
PolicyActionポリシー判定（Allow / Conditional / Review / Deny）
ReasonCode判定理由のコード（機械処理用）
AppliedPolicyName適用された組織ポリシー名
IsEffectiveルールが現在有効かどうか

Policy Settings & Judgments

Review OSS usage based on your organization's rules (v1.1.0)

Overview

Dependa's policy feature lets you define how your organization evaluates OSS dependencies, and records the judgment with its reasoning for review and sharing.

Move beyond detection — generate reports that explain why each decision was made.

Policy Builder

Open the Policy Builder from the "Policy" menu on the left sidebar.

Step 1: Choose a Template

Click "Apply Template" to load a pre-configured rule set.

TemplateDescription
BalancedGeneral-purpose risk balance. Recommended for most projects
StrictRestrictive settings for copyleft and unknown licenses
PermissiveRelaxed settings, suitable for internal tools

Step 2: Customize Rules

Adjust the action for each rule category.

Rule CategoryDescription
Strong copyleft licensesGPL / AGPL — source code disclosure obligations
Weak copyleft licensesLGPL / MPL — often usable as a separate library
Unknown licensePackages where the license could not be identified
Critical vulnerabilityCVSS 9.0+ known vulnerabilities
High vulnerabilityCVSS 7.0–8.9 known vulnerabilities
Low confidence classificationLicense detection is incomplete or inconsistent
Missing license metadataNo license metadata found for the package

Available actions for each rule:

ActionMeaningColor
Not AllowedUsage is not recommended by policyRed
Requires ReviewReview required before useOrange
ConditionalAllowed under certain conditionsBlue
AllowedNo restrictionsGreen

Step 3: Save and Set as Default

  1. Enter a policy name and click "Save"
  2. Click "Set as Default" to apply this policy automatically in future scans

If only one policy is saved, it will be automatically applied even without clicking "Set as Default".

Policy Judgments

When a scan completes, policy judgment records are automatically generated for each package.

The 5 Columns

ColumnDescription
PackagePackage name
Detected FactWhat was detected (e.g., License: Approved, Medium — CVE-2024-xxxx)
Initial StatusAutomatic evaluation by default rules (Pass / Warning / Fail)
Policy ActionOrganization policy judgment (Allow / Conditional / Review / Deny / No restriction)
ReasonWhy this judgment was made (e.g., Unknown license: Requires review by policy)

Reading the Policy Action Column

DisplayMeaning
DenyUsage is not recommended by the configured policy
ReviewReview required before use per policy configuration
ConditionalAllowed under conditions specified in the policy
AllowNo restrictions per policy
— (No restriction)Policy is applied, but no matching rule for this package
— (No policy)No organization policy has been configured

HTML Report

The HTML report generated from scan results includes a policy judgment section.

  • Summary: Applied policy name and action count badges
  • Detail table: Per-package facts, judgments, and reasons

Reports are self-contained HTML files that work offline and can be shared with team members and reviewers.

CSV Policy Columns

The CSV export (v2 schema) includes these additional columns:

ColumnDescription
PolicyActionPolicy judgment (Allow / Conditional / Review / Deny)
ReasonCodeMachine-readable reason code
AppliedPolicyNameName of the applied organization policy
IsEffectiveWhether the rule is currently enforced
