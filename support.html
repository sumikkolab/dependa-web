不具合の報告方法

不具合が発生した場合は、以下の手順で診断レポートを生成し、サポートにお送りください。

Step 1: 診断レポートを生成

アプリの 設定 画面の「サポート / トラブルシューティング」セクションにある 「診断レポートを生成」 ボタンをクリックしてください。ZIP ファイルが自動生成されます。

Step 2: 状況を記入

ZIP 内の user-message.txt を開き、再現手順・期待する動作・実際の動作を記入してください。

Step 3: メールで送信

生成された ZIP ファイルと、記入済みの user-message.txt を support@sumikkolab.com に添付して送信してください。

診断レポートについて

診断レポートには以下の情報が含まれます。

  • アプリのバージョン情報
  • OS・ランタイム環境情報
  • スキャン設定の状態（パスやファイル内容は含みません）
  • 設定ファイルの状態（存在確認・サイズ・更新日時のみ）

プライバシー: 診断レポートは、ソースコード、依存ファイル、プロジェクトパス、ユーザー名、マシン名を含まない設計になっています。

サポートデータの取り扱い

不具合調査のために提供されたデータは、調査目的以外に使用せず、調査完了後速やかに破棄するよう努めます。

How to Report Issues

If you encounter a problem, please generate a diagnostic report and send it to our support team.

Step 1: Generate a Diagnostic Report

Go to the Settings page in the app, find the "Support / Troubleshooting" section, and click "Generate Diagnostic Report". A ZIP file will be created automatically.

Step 2: Describe the Issue

Open user-message.txt inside the ZIP and fill in the reproduction steps, expected behavior, and actual behavior.

Step 3: Send via Email

Attach the generated ZIP file and the filled-in user-message.txt to an email and send it to support@sumikkolab.com.

About the Diagnostic Report

The diagnostic report includes:

  • App version information
  • OS and runtime environment details
  • Scan configuration status (no paths or file contents)
  • Settings file status (existence check, size, and last modified date only)

Privacy: The diagnostic report is designed not to include source code, dependency files, project paths, user names, or machine names. Please see the list above for what is included.

Data Handling

Data provided for investigation purposes will not be used for any other purpose and will be promptly deleted after the investigation is complete.
